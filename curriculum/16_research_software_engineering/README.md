# Module 16 — Research Software Engineering

**Tier:** 1 — Master  
**Week budget:** 4 weeks  
**Track A relevance:** RSE/Bioinformatics Engineer roles test software engineering discipline directly — packaging, testing, CI/CD, and documentation are core interview topics.  
**Track B relevance:** Supervisors expect PhD students to write reproducible, well-tested code; this module produces the compbio_utils package used across the curriculum.

## Objectives

By the end of this module you will be able to: structure a Python package; write and run a pytest suite; document code with NumPy-style docstrings + Sphinx; build a CLI tool with Click; configure GitHub Actions CI; profile Python code and identify bottlenecks; and publish a package to TestPyPI.

## Output artifact

`utilities/compbio_utils/` — the shared, tested, packaged Python utility library used by all prior modules.
