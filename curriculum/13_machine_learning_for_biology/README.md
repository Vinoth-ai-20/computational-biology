# Module 13 — Machine Learning for Biology

## Tier 1 — Master

**Week budget:** 8 weeks  
**Notebooks:** 15  
**Mini-projects:** MP01 (cancer classification), MP02 (drug-target binding prediction)

---

## Objectives

By the end of this module you will be able to:

1. Implement core supervised learning algorithms from scratch (linear/logistic regression, decision trees, SVM, k-NN) and explain every design choice
2. Apply the full ML pipeline: data cleaning → feature engineering → model selection → hyperparameter tuning → evaluation
3. Interpret models correctly in biological contexts (class imbalance, small n / large p, batch effects, data leakage)
4. Use scikit-learn fluently for standard tasks without needing to look up API details
5. Recognize when deep learning is and isn't needed (and distinguish Module 13 problems from Module 14 problems)
6. Explain, in an interview setting, how a given ML method works and why you chose it

---

## Why this helps Track A

Indian RA/JRF/Bioinformatics Engineer interviews routinely ask:
- "How does logistic regression work? What are its assumptions?"
- "You have 200 cancer samples and 50,000 genes. What is the curse of dimensionality and how do you handle it?"
- "What is cross-validation and why use it instead of a train/test split?"
- "What evaluation metric would you use for a highly imbalanced cancer classification problem?"

All of these are answered by this module. Building MP01 (cancer subtype classification) gives a concrete answer to "show me a ML project you built."

---

## Why this helps Track B

European PhD programmes in computational biology (EMBL EIPP, IMPRS, Wellcome Sanger) expect:
- Demonstrated ability to apply ML to biological data — not just use sklearn, but understand what's happening
- A portfolio project with real biological questions (MP01/MP02)
- Ability to read ML papers in biology critically (papers.md provides the foundational reading list)
