# Module 20 — Mini Projects

## MP01 — Portfolio publication

**Objective:** publish the computational-biology repository in a state that a PhD committee would find credible as a Track B artifact.

**Checklist:**
1. Root README.md describes all 20 modules with one-line summaries and depth tiers
2. ENVIRONMENT.md has a one-command bootstrap (conda env create -f environment.yml)
3. At least one module has a Zenodo DOI badge (connect GitHub repo to Zenodo, create a release)
4. Three capstone project notebooks run cleanly from a fresh conda environment
5. `portfolio/` directory contains 3 polished outputs (one per CP), each with an abstract + figure + README

**Deliverable:** the repository itself, in its published state, with a `portfolio/README.md` that links each CP output.

## MP02 — Track A interview simulation

**Objective:** prepare for the most common Track A bioinformatics interview pattern.

**Tasks:**
1. Pick 5 papers from the programme (one from Modules 8, 9, 10, 13, 14)
2. For each: practice cold explanation (3-minute limit; Module 19 NB10 protocol)
3. For each: write the 3-sentence explanation as a `paper-notes/` entry
4. Have a peer or Claude evaluate each explanation against the scoring rubric from Module 19 NB10

**Deliverable:** 5 `paper-notes/` entries with Pass-3 logged + cold explanation text.
