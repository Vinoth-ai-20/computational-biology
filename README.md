## Overview

This repository is my personal **Computational Biology Research Notebook**, designed as an intensive **12-month learning and portfolio-building program**.

Unlike a traditional university curriculum, this repository focuses on developing the knowledge, coding ability, research skills, and reproducible workflows required to become a Computational Biology researcher.

The primary goals are:

* Build a strong foundation in Computational Biology and related disciplines.
* Develop high-quality scientific software using Python.
* Learn through Jupyter notebooks, coding exercises, and mini research projects.
* Read, analyze, and reproduce influential research papers.
* Create a public portfolio demonstrating research readiness.
* Prepare for Computational Biology research roles in India.
* Prepare competitive applications for fully funded European PhD programs.

# Repository Philosophy

This repository follows five guiding principles.

### Notebook First

Every concept is learned through interactive Jupyter notebooks containing:

* Theory
* Mathematics
* Biological background
* Python implementation
* Visualizations
* Exercises
* Reflection
* References

---

### Learn by Building

Knowledge is developed through implementation rather than passive reading.

Every module includes:

* Coding exercises
* Assignments
* Mini projects
* Portfolio artifacts

---

### Research Driven

Every topic is connected to real Computational Biology research.

The curriculum emphasizes:

* Scientific computing
* Bioinformatics
* Systems Biology
* Machine Learning for Biology
* Research Software Engineering
* Reproducible research

---

### Portfolio Focused

Every completed module should produce something worth showcasing publicly.

Examples include:

* Jupyter notebooks
* Scientific visualizations
* Python packages
* Mini research projects
* Literature reviews
* Blog articles
* Open-source contributions

---

### Reproducible Science

All work should be reproducible.

The repository emphasizes:

* Git
* GitHub
* Version control
* Virtual environments
* Scientific documentation
* Clean code
* Reproducible notebooks
